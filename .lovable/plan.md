## 现状诊断

「缘分配对」目前的体感很像一份"心理学小报告"：
- 输入：双方姓名 + MBTI/星座下拉 + 一句描述。
- 加载：一颗旋转的爱心 + "分析中"。
- 结果：1 个百分比、5 维进度条、3-4 条优势、2 个冲突、爱之语、深度长文。

问题：
1. **没有故事感**：纯参数表 → 直接出报告，缺少"被命运抽中"的仪式感。
2. **结果同质化**：每次都是百分比+长段文字，缺少可截屏炫耀的"标签 / 人设 / 名场面"。
3. **不可分享**：socialCaption 是干文字，没有强视觉记忆点（Y2K 卡、CP 名、命运标签）。
4. **不互动**：用户被动看 AI 输出，没有"猜一下""选一下"的参与感。
5. **没有讨论钩子**：没有"红绿灯""避雷标签""命定关键词"这类年轻人乐意截图发评论区的素材。

## 改造目标
把它从「兼容度报告」升级成 **"命运盲盒抽卡 + CP 人设档案"**，单局体验 ≈ 抽卡 + 看八卦杂志 + 做心理测试。

## 新体验流程

### 1. 输入升级——"建档"仪式感
保留原有字段，但视觉重做：
- 顶部一条进度："Step 1/3 录入你 → Step 2/3 录入 TA → Step 3/3 摇命运骰子"。
- 新增 1 个**轻量趣味问题**（单选，预设 6-8 张"关系卡"）：
  - "你们最近一次互动是…" 选项：深夜长聊 / 已读不回 / 互发表情包 / 一起吃饭 / 还没说过话 / 刚吵完架。
- 新增"关系阶段"芯片：暗恋中 / 暧昧期 / 在一起 / 老夫老妻 / 复杂…（3 选 1）。
- 提交按钮文案改成 **"摇一摇命运骰子 🎲"**，按下时屏幕轻震 + 三秒倒数动画。

### 2. 加载升级——"宇宙正在算命"
30-50 步台词流式播报（前端定时器，3-5 秒）：
- "🌙 月亮正在比对你们的星图…"
- "💌 翻出你们前世的信件…"
- "⚖️ 称一下两颗心的重量…"
- "🎴 抽出今日命运卡…"
末尾"咔哒"开卡音效占位（用现有音效或省略）。

### 3. 结果升级——"CP 抽卡档案"

把单一长报告拆成 **6 张可滑动 / 可分享的卡片**，每张独立卡 = 独立分享素材：

**Card 1 · 命运卡（主卡）**
- 大字：CP 命名（例："风与月" / "猫与窗台"），AI 生成。
- 命运标签：3 个短词 chips（例："互补型""慢热但上头""注定要吵两次"）。
- 命运分数：保留 0-100% 但加一个**等级名**——
  - 90+ = "天选 SSR" / 75+ = "命定 SR" / 60+ = "缘分 R" / <60 = "练手 N"。
- 抽卡稀有度光效（SSR 金、SR 紫、R 蓝、N 灰）。

**Card 2 · 红绿灯**
- 🟢 绿灯 / 🟡 黄灯 / 🔴 红灯 三栏，每栏 1-2 条短句。
- 例：🟢 价值观对线｜🟡 情绪反应不同步｜🔴 都不主动认错。
- 这是 TikTok/小红书 最爆的格式之一。

**Card 3 · 五维雷达图**
- 现有 5 维改为**雷达图**（recharts 已在依赖里），更酷更可截屏。
- 雷达上方加一句 AI 生成的"一句话总评"。

**Card 4 · 名场面剧本**
- 标题"如果你们拍偶像剧"。
- 给出 1 段 80-120 字的小剧场（场景 + 对白 + 旁白）。
- 例："周五晚 11 点，你假装路过 TA 楼下…"
- 这是分享/转发率最高的内容。

**Card 5 · 爱之语 + 操作指南**
- 保留现有"我的爱之语 / TA 的爱之语"。
- 改成"💚 给 TA 的小动作"+"💛 TA 想从你这听到的话"两条具体可执行建议。

**Card 6 · 命定关键词 + 一句话占卜**
- 3 个未来关键词（例："旅行 / 误会 / 二次确认"）。
- 一句话占卜（例："立秋之前，你们会有一次重要的对话。"）。
- 长按可复制做星座号文案。

### 4. 分享升级——海报模板专属化
现有 `useSharePoster` 通用海报 → 为本测评做 **"塔罗式 CP 卡海报"**：
- 上半：CP 命名 + 稀有度徽章 + 命运分数。
- 中部：3 个命运标签 chips + 雷达缩略图。
- 底部：一句话占卜 + 二维码 / 域名。
- 配色随稀有度变化（SSR 金黑、SR 紫粉、R 蓝白）。
- 分享按钮做成 **"晒卡 📸"** 而不是普通"分享"，并加"复制 CP 文案"快捷按钮。

### 5. 讨论钩子——评论区诱饵
结果页底部加一条"**和 TA 一起测**"按钮：
- 复制一段邀请文案 + 结果链接到剪贴板：
  "我们的命运卡是【风与月 · SSR 92%】🎴 你来抽抽我们的红绿灯 → {link}"
- 鼓励两人对比，自然形成传播。

## 涉及改动（技术）

### A. Edge Function `assessment-compatibility/index.ts`
扩展 tool schema：
- `cpName`（CP 命名）
- `rarity`（SSR/SR/R/N 字符串，前端再上色）
- `tags: string[]`（3 个命运标签）
- `trafficLight: { green: string[]; yellow: string[]; red: string[] }`
- `dramaScene: string`（80-120 字小剧场）
- `loveLanguage` 增加 `actionForThem` / `phraseTheyWant`
- `keywords: string[]`（3 个）
- `prophecy: string`（一句占卜）
- 调整 prompt 让模型按"年轻人/小红书风"输出，避免说教；保留 `socialCaption` 兼容现有海报。

### B. 前端 `src/pages/CompatibilityFlow.tsx`
- 输入页：新增"最近互动""关系阶段"两组 chip 选择，提交时拼进 `myProfile.context` 一并发给后端。
- loading 步：替换为台词轮播组件（纯前端定时器，无后端依赖）。
- 结果页：用 6 张 motion 卡片堆叠 + 横向 snap 滑动；雷达图用 recharts。
- 稀有度色板写在 `index.css` 作为 token：`--rarity-ssr / sr / r / n` 渐变。
- "晒卡 / 和 TA 一起测"两个新 CTA。

### C. 海报 `useSharePoster` 调用
- 传入新字段（cpName、rarity、tags、prophecy），在 `bars` 之外加 `chips` 与 `prophecy` 区块。
- 若 `useSharePoster` 不够灵活，新增一个 `compatPoster` 专用模板（在 hook 内做 variant 分支）。

### D. i18n
- `assessmentFlow.compatibility.*` 下新增：rarity 名称、台词列表（loadingLines[]）、关系阶段 chips、最近互动 chips、红绿灯标题、剧场标题、占卜标题、晒卡 / 邀请文案。
- 中英两份。

### E. 数据兼容
- `compatibility_reports.result_data` 是 jsonb，多加字段不破坏现有数据。历史报告显示时若缺新字段，对应卡片做空态隐藏。

## 取舍
- **不动后端表结构**：所有新内容都塞进 `result_data` jsonb。
- **不引入新依赖**：recharts、framer-motion、lucide-react 都已存在。
- **优先视觉与分享物**，深度长报告保留作为最后一张可折叠卡，不再是主角。

## 范围确认
本次只重做「缘分配对」一项测评的输入 / 加载 / 结果 / 分享四段体验，不改其他测评、不改订阅与计费。
